# Handoff: Moneymon — cute financial-habit fairy-fountain game

## Overview
Moneymon is a mobile financial-habit game in the spirit of *Neko Atsume*. Players complete
real-world money habits ("quests") to earn a soft currency called **Wishes** (the heart-coin).
They **toss** Wishes into a fountain to summon collectible **fairies**, who visit for a limited
time, can be interacted with, drop crafting **materials**, and are recorded in a **Fairy Log**
(compendium / Pokédex equivalent). The fountain visually levels up as the player progresses.

The **Home / Fountain View** is the heart of the app — the screen players return to every session.

## About the Design Files
The single file in this bundle — `Moneymon Wireframes.html` — is a **design reference created in
HTML**. It is a gallery of lo-fi wireframes (9 mobile screens laid out side by side), **not
production code to copy directly**.

Your task is to **recreate these screens in the target codebase's environment** using its
established patterns and component library. If no environment exists yet, choose the most
appropriate stack for a mobile game-like app (e.g. React Native / Expo, Flutter, SwiftUI, or
React + a mobile web shell) and implement there. Do **not** ship the HTML file as-is.

The HTML file is self-contained (loads Google Fonts "Patrick Hand" + "Kalam"; all icons are inline
`<svg>` symbols; bottom nav and tweak controls are generated by a small `<script>` at the bottom).
Open it in a browser to see all screens and to use the tweak bar (glow color + ink weight) at the top.

## Fidelity
**Low-fidelity (lofi) wireframes.** These define **layout, hierarchy, flow, and content** — not final
visual design. The sketchy hand-drawn aesthetic (Patrick Hand font, wobbly borders, single yellow
highlighter accent) is a *wireframe convention to signal "not final"*, **not** the intended product
skin. Use these as a structural guide and apply the project's real design system / art direction for
styling. Treat exact px values, fonts, and the `#f3df8e` yellow as approximate placeholders.

The fairy/material artwork shown (egg silhouettes, hatched "leaf/wood/dew" boxes, the stick-figure
arrival character) are **placeholders** — real game art replaces them.

## Global Layout & Persistent Chrome
All screens are designed for a **portrait phone** (mock frame is 300×620 in the wireframe; build
responsive to the device).

- **Top bar (persistent):**
  - Top-left slot: context-dependent — a **Quests** entry icon (Home screens) OR a **Back** arrow
    (drill-in screens) OR a **screen title**.
  - Top-right: **Wish balance** — heart-coin icon + number (e.g. `500`). Always visible.
- **Bottom nav (persistent, 4 tabs):** **Finance · Inventory · Fairy Log · Me**, each an icon + label.
  Active tab is highlighted (filled accent chip behind the icon, bold label).
  - In the wireframe the nav is generated from one array — keep it a single source of truth.
  - **Quests is intentionally NOT in the bottom nav** — it's reached from the Home top-left icon.
- Some screens (Quests, Toss, Fairy arrival, Interaction sheet) replace or cover the standard bottom
  nav — see each screen below.

## Screens / Views

### 1. Home / Fountain View  *(the main hub)*
- **Purpose:** Default landing screen. Player sees their fountain (reflects current level), Wish
  balance, and routes to everything else.
- **Layout:** Top bar (Quests icon left, Wish balance right). Centered **fountain** = three stacked
  tiers, narrow→wide top-to-bottom (≈42% / 70% / 96% width), forming a tiered-fountain silhouette.
  **"Level N"** label below the fountain. Persistent bottom nav.
- **Two presented states:**
  - **Base:** plain fountain + "Level 10" label. Tapping the fountain (when no fairy present) opens
    the **Toss** screen.
  - **Actions:** fountain wrapped in a **yellow glow** (the higher the level, the more it grows and
    glows), with two buttons below: **`Enhance ↑`** (spend to level the fountain up) and
    **`Wish ♥`** (primary button → opens Toss).
- **Behavior:** Fountain size + glow intensity are a function of level. If a fairy is currently
  visiting, the fairy appears at the fountain and tapping it opens the **Fairy Interaction Sheet**
  instead of Toss.

### 2. Toss (Fountain Interaction)
- **Purpose:** Spend Wishes to summon a fairy. Reached by tapping the fountain or the **Wish** button.
- **Layout (top→bottom):** Back arrow + Wish balance. Smaller glowing fountain. An **amount selector**
  = stepper (`–`  heart-coin `100`  `+`) **and** a slider below it. A helper line ("Tossing 100 —
  better odds at higher tosses"). A **rarity-odds preview**: three rows (★ / ★★ / ★★★) each with a
  horizontal bar + percentage (e.g. 70% / 25% / 5%) that **update live with the chosen amount**.
  Full-width primary **`Toss`** button.
- **Behavior:** Confirming the toss deducts Wishes and triggers the **Fairy arrival** sequence (or a
  "a fairy is on their way" pending state). Higher toss amounts shift odds toward rarer fairies.

### 3. Fairy Arrival
- **Purpose:** Cutscene/dialogue when a tossed fairy shows up.
- **Layout:** Wish balance top-right with a **`Skip ▶|`** button beneath it. Large centered fairy
  figure (placeholder). A **dialogue box** at the bottom with a name tag (e.g. "Antony"), a line of
  dialogue ("The lord is here."), and a **▽ continue chevron** bottom-right. No bottom nav.
- **Behavior:** Tap to advance dialogue; Skip jumps to the end. Resolves to the fairy being present
  at the fountain on Home.

### 4. Fairy Interaction Sheet  *(bottom sheet / modal)*
- **Purpose:** Opens when tapping an active fairy at the fountain.
- **Layout:** Fountain dimmed behind a scrim; a **bottom sheet** slides up with a drag handle, then:
  - **Header row:** fairy portrait + **name** + **rarity** (★ count) + **"Visit ends in 2h 14m"** timer.
  - **Friendship:** "Friendship · Lv 3" label + a **progress bar** with `4 / 6` fill.
  - Primary **interaction button** ("Pat Antony") + cooldown note ("Next pat available in 5h") —
    interaction is allowed **once per X hours**.
  - **"Possible drops"**: row of material thumbnails (Wood, Leaf, Dewdrop).
- **Behavior:** Interacting raises friendship and can yield material drops; then the button enters
  cooldown. Visit timer counts down; when it hits 0 the fairy leaves.

### 5. Quests
- **Purpose:** Where Wishes are earned via real money habits.
- **Layout:** Back arrow + **"Daily Goals"** title + Wish balance. A vertical list of **quest cards**,
  each = a **reward chip** (heart-coin in a circle + reward number, e.g. 10 / 30 / 50) on the left and
  a **task description** on the right. At the bottom, a **tab switcher: `Daily` / `Weekly`** (replaces
  the standard bottom nav on this screen).
- **States:** incomplete card (outlined) vs **completed** card (dimmed background, strikethrough text,
  filled heart in the reward chip).
- **Behavior:** Tap a card to mark complete (honesty system — optionally show a quick verify/confirm
  prompt first). On completion play a **satisfying animation that deposits the coins into the top Wish
  balance** (reward chip → flies up to balance, balance increments).

### 6. Inventory / Materials
- **Purpose:** Materials collected from fairy drops.
- **Layout:** Back arrow + centered **"Inventory"** title + Wish balance. Vertical list rows: a square
  **material thumbnail** with a **`×N` quantity badge** (bottom-right corner), then the material
  **name** (e.g. "Leaf", "Wood", "Dewdrop") with a short description area. Bottom nav with **Inventory**
  active.

### 7. Fairy Log — Grid  *(compendium / Pokédex)*
- **Purpose:** Collection & progression overview.
- **Layout:** Back arrow + **"Fairy Log"** title (with book/butterfly icon) + Wish balance. A **2-column
  grid of fairy cells**. **Discovered** fairies show portrait + name (e.g. "Webster"). **Undiscovered**
  show an **egg silhouette + "? ? ?"** name. A **`Next ▷`** pagination button. Bottom nav with **Fairy
  Log** active.

### 8. Fairy Log — Detail
- **Purpose:** Full record for one fairy.
- **Layout:** A framed card with the fairy **name on a tab** at the top (e.g. "Snowball"). Left column:
  portrait + trait **chips** ("spring", "cute"). Right column: **"Rarity ★★★"**, **"Friendship lvl"**
  + progress bar (`4 / 6`), a meta/lore line ("Visits: 12 · "Hums old lullabies by the water.""). Below:
  **"Drops"** = material thumbnails (Wood, Leaf). Footer: **`◁ Back`** / **`Next ▷`** to page between
  fairies. Bottom nav with **Fairy Log** active.

### (Suggested, not yet drawn) Finance tab
The bottom nav includes a **Finance** tab (bar-chart icon) that has no wireframe yet. It's the intended
home for the actual money-tracking the quests reference (spending log, account balance, etc.). Flag this
with the product owner before building.

## Interactions & Behavior (summary)
- Home fountain tap → Toss (no fairy) or Interaction Sheet (fairy present).
- Wish button → Toss. Enhance → fountain level-up flow (confirm cost).
- Toss confirm → deduct Wishes → Fairy arrival → fairy present on Home.
- Quest complete → optional verify prompt → coin-fly-to-balance animation → balance increments.
- Interaction sheet "Pat" → friendship up + possible drop → cooldown timer.
- Visit timer counts down; fairy departs at 0.
- Compendium paginates; undiscovered fairies are silhouettes.

## State Management (needed data)
- `wishBalance` (number), `fountainLevel` (number).
- `quests`: `[{ id, period: 'daily'|'weekly', task, reward, completed }]`; daily/weekly reset timers.
- `activeFairy`: `{ id, name, rarity, visitEndsAt, friendshipLevel, friendshipProgress, lastInteractAt, possibleDrops[] }` or null.
- `inventory`: `[{ materialId, name, qty }]`.
- `compendium`: `[{ id, discovered, name, rarity, friendshipLevel, totalVisits, lore, drops[], traits[] }]`.
- `toss`: selected amount, derived rarity-odds table (function of amount).
- Transitions: toss → arrival → activeFairy set; interaction cooldown; visit expiry; quest completion → balance + animation.
